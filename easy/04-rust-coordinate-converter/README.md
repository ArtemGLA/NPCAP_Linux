# Задание 4: Coordinate Converter

**Уровень:** Лёгкий  
**Технологии:** Rust  
**Время выполнения:** 2-3 часа

## Описание

При работе с БПЛА часто нужно конвертировать координаты между различными системами. Наиболее распространённые:

- **WGS84** (широта/долгота) — используется в GPS
- **UTM** (Universal Transverse Mercator) — метрические координаты на плоскости

В этом задании вам нужно написать библиотеку для конвертации координат.

## Цель

Реализовать функции в файле `src/lib.rs`:

1. Конвертация WGS84 → UTM
2. Конвертация UTM → WGS84
3. Расчёт расстояния между точками (формула Хаверсина)
4. Расчёт азимута (направления) между точками

## Что такое UTM

UTM делит Землю на 60 зон по долготе (по 6°). Каждая зона имеет свою проекцию с координатами в метрах:
- **Easting** — координата на восток (центральный меридиан = 500000м)
- **Northing** — координата на север (экватор = 0 для северного полушария)
- **Zone** — номер зоны (1-60)
- **Hemisphere** — полушарие (N/S)

Пример: Москва (55.7558°N, 37.6173°E) → UTM 37N 413589E 6178456N

## Что нужно реализовать

```rust
pub struct WGS84 {
    pub lat: f64,  // широта в градусах (-90..90)
    pub lon: f64,  // долгота в градусах (-180..180)
}

pub struct UTM {
    pub easting: f64,   // метры на восток
    pub northing: f64,  // метры на север
    pub zone: u8,       // зона (1-60)
    pub hemisphere: Hemisphere,
}

pub enum Hemisphere {
    North,
    South,
}

impl WGS84 {
    /// Создать координату из градусов
    pub fn new(lat: f64, lon: f64) -> Result<Self, CoordinateError>;
    
    /// Конвертировать в UTM
    pub fn to_utm(&self) -> UTM;
    
    /// Расстояние до другой точки в метрах (формула Хаверсина)
    pub fn distance_to(&self, other: &WGS84) -> f64;
    
    /// Азимут (направление) к другой точке в градусах (0-360)
    pub fn bearing_to(&self, other: &WGS84) -> f64;
}

impl UTM {
    /// Конвертировать обратно в WGS84
    pub fn to_wgs84(&self) -> WGS84;
}
```

## Формулы

### Формула Хаверсина (расстояние)

```
a = sin²(Δφ/2) + cos(φ1) * cos(φ2) * sin²(Δλ/2)
c = 2 * atan2(√a, √(1-a))
d = R * c

где:
  φ — широта в радианах
  λ — долгота в радианах
  R — радиус Земли (6371000 м)
```

### Азимут (bearing)

```
θ = atan2(sin(Δλ) * cos(φ2), cos(φ1) * sin(φ2) - sin(φ1) * cos(φ2) * cos(Δλ))
```

### UTM конвертация

UTM использует проекцию Transverse Mercator. Упрощённые формулы:

```
k0 = 0.9996  // масштабный коэффициент
e = 0.0818191908426  // эксцентриситет эллипсоида WGS84
a = 6378137  // большая полуось WGS84 в метрах

zone = floor((lon + 180) / 6) + 1
λ0 = (zone - 1) * 6 - 180 + 3  // центральный меридиан зоны

// Далее применяются формулы проекции...
```

## Примеры

```rust
use coordinate_converter::{WGS84, UTM, Hemisphere};

// Москва
let moscow = WGS84::new(55.7558, 37.6173).unwrap();

// Конвертация в UTM
let utm = moscow.to_utm();
assert_eq!(utm.zone, 37);
assert!(matches!(utm.hemisphere, Hemisphere::North));

// Обратная конвертация
let back = utm.to_wgs84();
assert!((back.lat - moscow.lat).abs() < 0.0001);

// Расстояние до Санкт-Петербурга
let spb = WGS84::new(59.9343, 30.3351).unwrap();
let distance = moscow.distance_to(&spb);
assert!((distance - 635_000.0).abs() < 10_000.0);  // ~635 км

// Азимут
let bearing = moscow.bearing_to(&spb);
println!("Направление на СПб: {:.1}°", bearing);  // ~318°
```
